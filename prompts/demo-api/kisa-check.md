Use `gateproof:kisa-check` on `examples/demo-api/` and produce a markdown report.

Requirements:

- optimize for Korea-oriented KISA-style readiness review
- use pass, partial, fail style judgement where appropriate
- call out evidence gaps separately from confirmed failures
- prioritize admin exposure, secret handling, logging, authentication, and baseline hardening
- include concrete evidence references to files when possible
- avoid claiming confirmed runtime exploitation without evidence

Expected output shape:

1. Summary
2. Critical gaps
3. Findings by control area
4. Evidence seen
5. Missing evidence
6. Recommended next steps
