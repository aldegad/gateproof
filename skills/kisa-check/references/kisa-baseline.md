# KISA Baseline

Use this baseline for Korea-oriented compliance readiness reviews.

## Core domains

1. Account and authentication controls
2. Privilege management
3. Access control and network restriction
4. Encryption in transit and at rest
5. Logging and auditability
6. Secure configuration and hardening
7. Patch and vulnerability management
8. Operational evidence and policy support

## Review posture

- Prefer evidence over assumption.
- If proof is missing, record it as missing evidence instead of silently passing the item.
- Separate "control not implemented" from "control may exist but was not evidenced."

## Judgement model

- Pass: sufficient evidence and acceptable implementation
- Partial: some implementation exists but is incomplete or weak
- Fail: missing, unsafe, or clearly non-compliant
- N/A: genuinely out of scope
